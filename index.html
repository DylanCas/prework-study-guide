HTML

  • The head element contains info about the webpage, for the browser.
  • The body element contains all the visible content on a webpage.
  • The Meta element item, 'UTF-8', is the most common for use in webpages.
  • Elements nested within other elements are called child elements, within the parent elements.
  • All content in a webpage is closed within "tags", usign the "<" and ">" symbols.

CSS

  • There are 3 ways to style a page using CSS:
  1. Inline CSS - Apply CSS style rules to a specific HTML tag. Useful for quick changes, but can only change one element at a time.
  2. Internal CSS style sheet - Can embed a style sheet nesting a "style" element within the "head" element.
  3. External CSS style sheet - Most common as it allows all the CSS rules to stay within a separate file. File is linked using a "link" element in the "head" element.

  • CSS dictates the colors, visual layout, and organization of the the page.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS uses "class" and "element" selectors to assist with formatting.
  • Class selector will apply styles to certain selected elements without affecting ALL of that element type.

Git

Git Flow

  1. git add -A - Stages all changes that occured within the working branch.
  2. git commit -m "commit message" - Commits new entries/work to the GitHub repo, with a short descriptive message.
  3. git pull origin main - Once changes are added and committed locally, makes sure local branch is in sync with our working feature branch.
  4. git push origin branch name - Pushes commited changes to the remote repo. Usually ready to perform "Pull Request" and confirm "Merge" after this, within GitHub,

  • git status - Checks what branch we are currently on
  • git checkout -b branch-name - Creates a new branch and switches to it

JavaScript

JavaScript Types:

  • Primitive values - All types except objects define "immutable values", values which can't be changed.
  • Boolean type - Represents a logical entity that can have two values: true and false.
  • Null type - Has exactly one value: null. Represents the intentional absence of any object value. Treated as 'false' for boolean operations.
  • Undefined type - A variable that has not been assigned a value has the 'undefined'.
  • Numeric types - ECMAScript has two build in numeric types: Number and BigInt - along with the related value NaN.
  • Number type - Double-precision 64-bit binary format IEEE 754 value.
  • BigInt type - A numeric primitive in JavaScript that can represent integers with arbitrary precision. With BigInts, you can safely store and operate on large integers even beyond the safe integer limit for numbers.
  • NaN - "Not a Number", typically encountered when the result of an arithmetic operation cannot be expressed as number. It is also the only value in JavaScript that is not equal to itself.
  • String type - Used to represent textual data. It is a set of "elements" of 16-bit unsigned integer values. Each element in the string occupies a position in the string. The first element is at index 0, next at index 1, and so on. The length of a string is the number of elements in it.
  • Symbol type - A Symbol is a unique and immutable primitive value and may be used as the key of an object property. In some programming languages, symbols are called "atoms".
  • Objects - In Comp Sci, an object is a value in memory which is possibly referencede by an identifier.

  • Order matters when using JavaScript. The script file should load top to bottom, as that is the way it is read. However, certain conditional statements can alter or interrupt this flow somewhat.
